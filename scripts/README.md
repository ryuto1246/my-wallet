# CSV インポートスクリプト

Google スプレッドシートからエクスポートした CSV ファイルを Firestore にインポートするためのスクリプトです。

## 前提条件

- Firebase Admin SDK のサービスアカウントキーが必要です
- CSV ファイルが正しいフォーマットであること

### サービスアカウントキーの取得方法

1. [Firebase Console](https://console.firebase.google.com/)にアクセス
2. プロジェクトを選択
3. 設定（⚙️）→ プロジェクトの設定 → サービスアカウント
4. 「新しい秘密鍵を生成」ボタンをクリック
5. ダウンロードした JSON ファイルをプロジェクトルートに `serviceAccountKey.json` として保存

**重要**: `serviceAccountKey.json` は機密情報です。`.gitignore` に追加して、Git にコミットしないでください。

## 使い方

### 1. CSV ファイルの準備

CSV ファイルは以下のフォーマットである必要があります：

```csv
日付,取引コード,振替先コード,カテゴリーコード,金額,メモ,カテゴリー,取引種類,支払い手段,立替区分,振替先
10/25,300,,,¥0,,,残高確認,現金,,
10/26,030,,702,¥3781,APPLE PENCIL 分割払い,学業教養/文房具,支出,三井住友カード,,
```

### 2. スクリプトの実行

以下のコマンドでインポートを実行します：

```bash
npm run import-csv -- <UID> <CSVファイルパス>
```

#### 例

```bash
npm run import-csv -- kiEVl0vpLpPEMm17qFVvflAqUvx2c "家計簿 - 取引記録.csv"
```

または、直接 tsx で実行することもできます：

```bash
npx tsx scripts/import-csv.ts kiEVl0vpLpPEMm17qFVvflAqUvx2c "家計簿 - 取引記録.csv"
```

### 3. パラメータ

- `<UID>`: Firebase のユーザー ID
- `<CSVファイルパス>`: インポートする CSV ファイルのパス

## データマッピング

### 支払い手段

CSV の支払い手段は以下のようにマッピングされます：

| CSV             | Firestore |
| --------------- | --------- |
| 現金            | cash      |
| ソニー銀行      | sony_bank |
| 三井住友カード  | olive     |
| 三井住友銀行    | smbc_bank |
| OLIVE           | olive     |
| paypay / PayPay | paypay    |
| d 払い          | d_payment |
| d カード        | d_card    |

### カテゴリー

CSV のカテゴリー（例：`食費/外食`）は以下のようにマッピングされます：

| CSV                   | main（メイン） | sub（サブ）          |
| --------------------- | -------------- | -------------------- |
| 収入/アルバイト       | 収入           | アルバイト・給与     |
| 食費/外食             | 食費           | コンビニ             |
| 交通費/電車バス       | 交通費         | 電車・バス（定期外） |
| 学業教養/ソフトウェア | 教育・教養     | ソフトウェア・ツール |
| ...                   | ...            | ...                  |

詳細なマッピングは `scripts/import-csv.ts` の `CATEGORY_MAP` を参照してください。

### 立替区分

立替情報は以下のように処理されます：

#### 1. 支出の立替（立替区分がある場合）

| CSV      | Firestore | 説明                           |
| -------- | --------- | ------------------------------ |
| 親       | parent    | 親のために立て替えた（未回収） |
| 友人     | friend    | 友人のために立て替えた         |
| Nectere  | work      | 仕事関連で立て替えた           |
| 東大吹部 | work      | 団体のために立て替えた         |

→ `advance: { isPaid: false, advancedFor: "parent", note: "親" }`

#### 2. 収入の立替金回収（カテゴリーが「収入/立替金回収」の場合）

立替区分は空欄でも、カテゴリーから自動判定します。

→ `advance: { isPaid: true, advancedFor: "other", note: "立替金回収" }`

### スキップされるレコード

以下のレコードはインポートされません：

- 取引種類が「残高確認」のもの
- 金額が 0 円のもの

## 注意事項

- インポートは追加のみで、既存のデータは削除されません
- 重複チェックは行われないため、同じ CSV を複数回実行すると重複してインポートされます
- エラーが発生した行はスキップされ、処理が続行されます
- インポート結果はコンソールに表示されます

## トラブルシューティング

### Firebase 設定エラー

```
Firebase設定が不完全です。.env.localファイルを確認してください。
```

→ `.env.local` ファイルに以下の環境変数が設定されているか確認してください：

- `NEXT_PUBLIC_FIREBASE_API_KEY`
- `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN`
- `NEXT_PUBLIC_FIREBASE_PROJECT_ID`
- `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET`
- `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID`
- `NEXT_PUBLIC_FIREBASE_APP_ID`

### CSV ファイルが見つからない

```
CSVファイルが見つかりません: <ファイルパス>
```

→ ファイルパスが正しいか確認してください。相対パスまたは絶対パスで指定できます。

### パースエラー

特定の行でエラーが発生した場合、その行の情報とエラー内容がコンソールに出力されます。
CSV のフォーマットが正しいか確認してください。
