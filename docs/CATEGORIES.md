# 📂 カテゴリー構造ガイド

## 概要

このドキュメントでは、Smart Wallet アプリのカテゴリー構造について説明します。カテゴリーは AI サジェスチョンの精度向上と、詳細な支出分析のために設計されています。

## カテゴリー統計

- **メインカテゴリー**: 14 個（収入 1 個、支出 13 個）
- **サブカテゴリー**: 100 個以上
- **詳細度**: 用途・時間帯・場所に応じた細分化

## AI サジェスチョンとの連携

### キーワードベースの推測

AI は以下のようなキーワードを認識して、適切なカテゴリーを提案します：

**食費関連**

- `スーパー` → 自炊（食材購入）
- `コンビニ` → コンビニ
- `ランチ` → 外食（ランチ）
- `スターバックス` → カフェ・喫茶
- `飲み会` → 飲み会・宅飲み / 交際費（飲み会）

**交通費関連**

- `電車`, `バス` → 電車・バス（定期外）
- `定期` → 定期券
- `タクシー` → タクシー・代行
- `ガソリン` → 車（ガソリン・駐車場）

**その他**

- `バイト`, `給料` → 収入（アルバイト・給与）
- `Netflix` → 通信・サブスク（動画配信）
- `映画` → 趣味・娯楽（映画・演劇）

### コンテキスト学習

AI は以下のコンテキスト情報を活用して精度を向上させます：

1. **時間帯**:

   - 昼食時 → 外食（ランチ）、学食
   - 夕食時 → 外食（ディナー）、飲み会・宅飲み

2. **曜日**:

   - 平日の移動 → 定期券の可能性
   - 週末の移動 → 電車・バス（定期外）の可能性

3. **金額**:

   - 300-600 円 → コンビニ、カフェ・喫茶
   - 1000-2000 円 → 外食（ランチ）
   - 3000 円以上 → 外食（ディナー）、飲み会

4. **決済方法**:
   - 現金 → コンビニ、学食の可能性
   - クレカ → サブスク、定期券の可能性

### 学習機能

ユーザーが AI の提案を修正すると、以下の情報が学習されます：

- 特定のテキスト入力に対する正しいカテゴリー
- 時間帯・曜日とカテゴリーの関連性
- 金額帯とカテゴリーの関連性
- 類似したパターンの推測

## 使い方のヒント

### 効果的なカテゴリー選択

1. **詳細度を活用する**

   - ❌ 悪い例: すべての食事を「食費 - その他」に分類
   - ✅ 良い例: 購入場所や時間帯で細かく分類

2. **一貫性を保つ**

   - 同じ店舗・同じ用途は同じカテゴリーに分類
   - AI が学習してより正確な提案をしてくれます

3. **時間帯を考慮する**

   - 昼のレストラン → 外食（ランチ）
   - 夜のレストラン → 外食（ディナー）

4. **用途で判断する**
   - カフェで仕事 → 教育・教養（ソフトウェア・ツール）
   - カフェでリラックス → 食費（カフェ・喫茶）

### 分析のメリット

詳細なカテゴリー分類により、以下の分析が可能になります：

- 外食（ランチ）と外食（ディナー）の比率
- 自炊と外食のバランス
- 定期券と定期外交通費の割合
- サブスクの総額把握
- 時間帯別の支出傾向

## カテゴリー管理 API

### ヘルパー関数

```typescript
import {
  getMainCategories,
  getSubCategories,
  isValidCategory,
  getCategoryInfo,
  getRelatedSubCategories,
  suggestCategoriesByKeyword,
} from "@/constants/categories";

// メインカテゴリー一覧を取得
const mainCategories = getMainCategories(); // すべて
const incomeCategories = getMainCategories(true); // 収入のみ
const expenseCategories = getMainCategories(false); // 支出のみ

// サブカテゴリー一覧を取得
const foodSubs = getSubCategories("食費");

// カテゴリーの検証
const isValid = isValidCategory("食費", "自炊（食材購入）"); // true

// カテゴリー情報を取得
const info = getCategoryInfo("食費");
// { main: '食費', subs: [...], isIncome: false }

// 関連サブカテゴリーを取得
const related = getRelatedSubCategories("食費", "自炊（食材購入）");
// ['コンビニ', '外食（ランチ）', ...]

// キーワードから推測
const suggestions = suggestCategoriesByKeyword("スターバックス");
// [{ main: '食費', sub: 'カフェ・喫茶', isIncome: false }]
```

## 項目名のテンプレート

### 構造化された項目名

AI サジェスチョンでは、カテゴリーに応じた構造化された項目名を提案します。

#### テンプレート例

**食費系**

- スーパー: `スーパー（オオゼキ）で食材購入`
- コンビニ: `コンビニ（セブンイレブン）で購入`
- 外食: `レストラン（サイゼリヤ）でランチ`
- カフェ: `カフェ（スターバックス）でコーヒー`

**交通費系**

- 電車: `電車（JR）：渋谷→新宿`
- タクシー: `タクシー：渋谷→自宅`
- 定期券: `定期券（JR 渋谷〜新宿）購入`

**通信・サブスク系**

- `Netflix 月額料金`
- `Spotify Premium`
- `携帯料金（docomo）`

**その他**

- 衣類: `ユニクロでTシャツ購入`
- 娯楽: `映画（作品名）鑑賞`
- 収入: `アルバイト（◯◯カフェ）給与`

### テンプレートのルール

1. **店舗名・固有名詞は括弧で囲む**

   - 例: `スーパー（オオゼキ）`、`カフェ（スターバックス）`

2. **移動系は「：」と「→」を使用**

   - 例: `電車（JR）：渋谷→新宿`

3. **月額サービスは「月額」を明記**

   - 例: `Netflix 月額料金`

4. **具体的な商品名や用途を含める**

   - 例: `ユニクロでTシャツ購入`

5. **20 文字以内で簡潔に**

これにより、項目名が統一され、後から見返しても何の支出か一目で分かるようになります。
